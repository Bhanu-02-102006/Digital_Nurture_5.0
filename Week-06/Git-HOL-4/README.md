# Git Merge Conflict Resolution Hands-On Lab

This exercise demonstrates how to resolve merge conflicts in Git.

## Objectives

- Create a feature branch
- Modify the same file in different branches
- Generate a merge conflict
- Resolve the conflict manually
- Commit the resolved changes
- Ignore backup files using `.gitignore`

## Files

- `hello.xml` – XML file used to create and resolve the merge conflict.
- `.gitignore` – Ignores backup (`*.bak`) files.

## Branch Used

- GitWork

## Outcome

Successfully created a merge conflict, resolved it manually, committed the resolved file, updated `.gitignore`, and merged the branch into the main branch.
